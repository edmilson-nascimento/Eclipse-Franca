ABAP Development Tools for Eclipse

For an overview of new features for the ABAP core development, ABAP Core Data Services (CDS) development, and the usage of tools for integration of native SAP HANA objects, see the following sections:

 Simplified Logon for ABAP Cloud Projects

You now only need to enter the ABAP service instance URL when creating an ABAP cloud project in the context of SAP BTP ABAP environment.

The creation wizard is now reduced to one input field and can be used for both, SAP S/4HANA Cloud ABAP Environment and SAP BTP ABAP environment. This means the process is now the same like in the context of SAP S/4HANA Cloud ABAP Environment and you no longer need a service key.

For more information, see Creating an ABAP Cloud Project.

ABAP Debugger: Formatting Variables Type of JSON Format

In the Variables view, you can now also format and display variable types of JSON.

For more information, see Displaying and Formatting Types of Formats, Data and Programming Languages.

Displaying Field Characteristics for RAP Types in the Element Information Popup

The Element Information popup (F2) for RAP types shows additional field characteristics information such as mandatory:create, mandatory:save, or readonly. The displayed field characteristics are organized in a table and vary based on the RAP operation type.

Note

Fields that have behavior characteristics determined at runtime, such as feature:instance fields, are displayed in a Dynamic column.

Example of Element Information View for Specific RAP Types

For more information, see RAP - FieldCharacteristics.

Triggering Where-Used List for Usages of CDS Annotations

You can now also trigger a where-used search for CDS annotations. This enables you to investigate in which ABAP CDS objects an annotation is used.

In a CDS object, select the annotation and choose Get Where-Used List from the context menu. Alternatively, you can choose Ctrl + Shift + G . From the Get Where-Used List pre-execution dialog you have several possiblities to limit your search.

The result is displayed in the Search view. From the integrated toolbar, you can filter and change how the result is displayed. By double-clicking the finding, you can navigate to the relevant usage.

Rerunning Unit Tests with Coverage

You can now rerun unit tests with coverage in the ABAP Coverage view.

For more information, see Evaluating ABAP Unit Code Coverage Results.

Clearing Coverage Results

You can now clear your coverage results in the ABAP Coverage view.

For more information, see Evaluating ABAP Unit Code Coverage Results.

Working with Launchpad App Descriptor Items

You can now create launchpad app descriptor items that are required to call applications from the SAP Fiori launchpad.

For more information, see Working with Launchpad App Descriptor Items.

Working with Logical External Schemas

You can now create logical external schemas that can be used in combination with external entities for establishing outbound SQL access.

For more information, see Working with Logical External Schemas.

Displaying the Original WSDL in a SOAP Provider Model

You can now display the original WSDL file that you used to create a SOAP provider model.

For more information, see Creating SOAP Service Providers.

Multi-Dimensional Analysis App Generator

You can now use the multi-dimensional analysis app generator wizard to create all ABAP development objects that are needed to build, expose, and deploy a multi-dimensional analysis Fiori app.

For more information, see Generating Multi-Dimensional Analysis Applications.

New Class Interface for Application Job Catalog Entries and Templates

You can now work with a new attribute-based class interface. With the new class interface, you can now change the parameters, sections, and groups of a catalog entry. Value helps are now supported and can be selected in the catalog entry parameters. You can also run a class for testing in the application job template.

For more information, see Working with Application Job Objects.
